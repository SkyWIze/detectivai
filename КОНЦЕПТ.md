# 🕵️ «Detective» — концепт Telegram-бота

> Челлендж: написать ТГ-бота за неделю. Без ограничений, будет оцениваться жюри.
> Жанр: детектив с ИИ-мастером. Дата фиксации: 2026-06-06.

---

## 1. Видение

Telegram-бот, в котором игрок расследует уникальное убийство: выбирает вселенную,
**допрашивает подозреваемых живым текстом**, свободно осматривает локации, ловит
подозреваемых на лжи (предъявляя найденные улики) и в финале называет убийцу с
обоснованием. Каждое дело генерит LLM → игра бесконечно реиграбельна. Дёшево по
токенам за счёт архитектуры «состояние вместо истории диалога».

**Суть в одном предложении:** тебе дают дело об убийстве, ты допрашиваешь
подозреваемых своими словами, ищешь нестыковки, собираешь улики и называешь убийцу.

---

## 2. Зафиксированные решения

| Параметр        | Решение |
|-----------------|---------|
| Допрос          | Свободный текст |
| Дела            | Генерит LLM на лету |
| Формат          | Одиночка в личке |
| Сеттинг         | Микс: игрок выбирает (Классика / Нуар / Абсурд) |
| Улики           | Свободный осмотр локаций текстом |
| Победа          | Назвать убийцу **+ письменное обоснование** (оценивает LLM) |
| Противоречия    | Игрок сам предъявляет улику и «давит» на ложь |
| Ресурс          | Лимит ходов, ранг за эффективность |
| Ход тратят      | Только крупные действия (осмотр локации / раунд допроса) |
| Провал          | Одно обвинение → финал, показываем разгадку |
| Сохранения      | Дело живёт в SQLite, можно вернуться |
| Онбординг       | Короткий туториал первым делом |
| Размер дела     | 4 подозреваемых, 3–4 локации, ~15 мин |
| Подсказки       | Кнопка «💡», снижает ранг |
| Блокнот         | Авто: улики + кто допрошен + замеченные противоречия |
| Стретч-вау      | Дело дня + лидерборд; атмосферные арты/звук |

## 3. Полный пользовательский поток

```
/start
 └─ Новый игрок? → ТУТОРИАЛ (простое заготовленное мини-дело с подсказками по UI)
 └─ Иначе → Главное меню

Главное меню: [🔎 Новое дело] [▶️ Продолжить] [📊 Профиль/ранги] [🏆 Дело дня*]

Новое дело → выбор вселенной: [🎩 Классика] [🌃 Нуар] [🤡 Абсурд]
 └─ Генерация дела (2 LLM-запроса: создать → валидировать) → сохранение в БД
 └─ БРИФИНГ: «Найден труп… Вот 4 подозреваемых. У тебя N ходов.»

ИГРОВОЙ ЭКРАН (постоянная клавиатура):
 [👤 Допросить] [🔍 Осмотреть локацию] [📋 Блокнот] [💡 Подсказка] [⚖️ Обвинить]

— 👤 Допросить → выбор подозреваемого → свободный диалог (тратит 1 ход за «раунд»)
      внутри: пишешь вопросы; чтобы предъявить улику — кнопка [🎴 Предъявить улику]
— 🔍 Осмотреть → выбор локации → пишешь «осмотреть ящик стола» → LLM решает, что нашёл
— 📋 Блокнот → улики / допрошенные / замеченные противоречия / осталось ходов
— ⚖️ Обвинить → выбор убийцы + текст обоснования → ФИНАЛ

ФИНАЛ: правда раскрыта (убийца, мотив, орудие) + вердикт по твоему обоснованию
        + ранг сыщика + статистика + [Сыграть ещё] [В меню]
```

## 4. Модель данных

### `case` — само дело (генерится 1 раз, скрыто от игрока)

```json
{
  "id": "uuid", "setting": "noir",
  "title": "Смерть в баре «Полночь»",
  "victim": {"name": "Лео Маркс", "found": "за стойкой, удар по голове, 23:40"},
  "murderer_id": "s3",                         // СЕКРЕТ
  "weapon": "бутылка виски", "motive": "карточный долг", "time": "23:30",
  "suspects": [
    {"id":"s1","name":"...","persona":"нервный бармен",
     "alibi":"мыл стаканы в 23:30",
     "secret":"воровал из кассы",               // врёт, но НЕ убийца
     "knows":["видел, как s3 заходил в подсобку"],
     "lies_about":"время своего ухода",
     "breaks_on":["c2"]}                         // какая улика его колет
  ],
  "locations": [
    {"id":"l1","name":"подсобка","searchables":[
        {"desc":"ящик стола","clue":"c1"},
        {"desc":"мусорное ведро","clue":null}]}  // часть мест — пустышки
  ],
  "clues": [
    {"id":"c1","text":"расписка о долге Маркса на имя s3","implicates":"s3"},
    {"id":"c2","text":"касса недосчитывается 200$","implicates":"s1"}
  ],
  "solution_chain": "c1 (долг s3) + c3 (отпечаток на бутылке) + ложь s3 про время = s3 убийца"
}
```

### `session` — прогресс игрока (ТОЛЬКО это уходит в LLM при допросе)

```json
{
  "user_id": 123, "case_id": "uuid", "status": "active",
  "moves_left": 12,
  "interrogated": ["s1"], "found_clues": ["c1"],
  "noted_contradictions": [{"suspect":"s1","note":"алиби не сходится с кассой"}],
  "hints_used": 1,
  "suspect_dialogs": { "s1": [ {"q":"...","a":"..."} ] }   // короткая память на подозреваемого
}
```

## 5. Как работает «мозг» — LLM-взаимодействия

Четыре с половиной разных типа вызова, у каждого свой промпт и задача:

**A. Генерация дела** (1 раз на старте)
- Вход: выбранный сеттинг + жёсткая JSON-схема + правила («ровно 4 подозреваемых;
  у каждого секрет; убийца ровно один; цепочка улик ведёт к нему однозначно; часть
  мест осмотра — пустышки»).
- Выход: объект `case`.

**B. Валидатор дела** (1 раз, сразу после A)
- Отдельным запросом: «по `solution_chain` и доступным `clues` — однозначно ли
  выводится `murderer_id`? Нет ли дыр?». Если «нет» → регенерим (макс 2 попытки).

**C. Отыгрыш подозреваемого** (на каждый раунд допроса)
- В модель уходит **только кусок дела этого подозреваемого** (`persona`, `alibi`,
  `secret`, `knows`, `lies_about`) + последние реплики из `suspect_dialogs[id]` +
  (если игрок предъявил улику) текст улики.
- Жёсткие правила системника:
  - «Ты НЕ знаешь, кто убийца, и не знаешь чужих секретов.»
  - «Держись `persona`. Про `lies_about` — уверенно лги.»
  - «Колешься ТОЛЬКО если предъявлена улика из твоего `breaks_on`. Тогда частично
    признаёшься (но не выдаёшь убийцу, если это не ты).»
  - «Не выходи из роли, оффтоп мягко возвращай к делу.»
- Ключ к дешевизне и честности: подозреваемый физически не знает разгадки.

**D. Осмотр локации** (на каждый осмотр)
- Вход: `searchables` текущей локации + что игрок написал.
- Модель сопоставляет запрос с `searchables`: попал в `clue` → выдаём улику и пишем
  в `found_clues`; попал в пустышку → атмосферное «ничего важного».

**E. Судья финала** (1 раз)
- Вход: `solution_chain` + выбранный убийца + текст обоснования.
- Выход: верно/неверно + насколько обоснование совпало с реальной логикой → ранг.

## 6. Экономика и ранги

- **Ходы:** старт ~12 (осмотр локации и раунд допроса = −1; предъявление улики и
  чтение блокнота — бесплатно).
- **Ранг** считаем формулой (не через LLM — дёшево и предсказуемо):
  `база за верное обвинение + бонус за качество обоснования (судья E) + бонус за остаток ходов − штраф за подсказки`.
- Шкала: **Стажёр → Лестрейд → Мегрэ → Пуаро → Холмс**.
- Неверное обвинение → ранг «Дело провалено», но всегда показываем красивую разгадку.

## 7. Технический стек и структура

```
detective-bot/
├─ bot/            # aiogram 3: хендлеры, FSM-состояния, клавиатуры
│  ├─ handlers/    # start, game, interrogate, search, accuse
│  └─ keyboards.py
├─ core/
│  ├─ llm.py       # обёртка над провайдером (легко свапнуть Groq↔OpenAI)
│  ├─ case_gen.py  # генерация + валидация дела (A, B)
│  ├─ interrogate.py # отыгрыш подозреваемого (C)
│  ├─ search.py    # осмотр локаций (D)
│  ├─ judge.py     # финал + ранг (E)
│  └─ scoring.py
├─ db/             # SQLite: users, cases, sessions, leaderboard
├─ prompts/        # все системные промпты отдельными файлами
└─ config.py
```

- **FSM-состояния:** `menu → choosing_setting → briefing → playing →
  interrogating(suspect_id) → searching(location_id) → accusing → finished`.
- **LLM:** старт на Groq / Gemini Flash (бесплатно, быстро), переключатель на
  `gpt-4o-mini` через конфиг для финального качества.
- **БД:** SQLite (дела, сессии, лидерборд).
- **Деплой:** на демо — polling с ПК; дальше — VPS + webhook.

### Ориентир по стоимости
- Полностью бесплатный стек (Groq + SQLite) — 0 ₽, годится для демо.
- На `gpt-4o-mini`: одна партия ≈ $0.01–0.03; сотня тестовых партий ≈ $1–3.

## 8. Стретч-фичи (если останется время)

1. **Дело дня + лидерборд** — генерим одно общее дело в сутки (cron), все играют его,
   рейтинг по рангу/ходам. Переиспользует всю готовую механику. *Делать первой.*
2. **Атмосфера** — арты места преступления / портреты подозреваемых (генерация
   картинок) и/или эмодзи-звук в брифинге. Косметика поверх готового, отрезается без боли.

Не делаем в v1: голос, мультиплеер-кооп (в дальний бэклог).

## 9. Главные риски и митигации

| Риск | Митигация |
|------|-----------|
| Подозреваемый сливает убийцу | Ему не даём знать разгадку — только свой кусок дела (вызов C) |
| Дело нерешаемо | Валидатор (вызов B) + регенерация |
| LLM путается в показаниях | Фиксированный `case` в БД + свой `alibi/lies_about` + короткая память диалога |
| Игрок ломает оффтопом | Правило «оставаться в роли» в системнике |
| Дорого по токенам | В модель уходит state и кусок дела, а не вся история; ход тратят только крупные действия |
| Жюри не поймёт, как играть | Туториальное мини-дело + постоянная клавиатура с понятными кнопками |

## 10. Изюминки — то, что цепляет жюри

Фишки, которыми «обычный детектив-чат» не обладает. Все дешёвы по токенам.

### В v1 (ядро отличия)

1. **Перекрёстный допрос.** Подозреваемого можно колоть не только уликой, но и
   **показаниями другого**: «А Анна сказала, что видела тебя в подсобке в 23:30».
   Реализуется почти даром — мы и так храним `suspect_dialogs`; в вызов C добавляем
   опциональный блок «противоречащая цитата другого свидетеля». Алиби начинают
   рассыпаться → настоящая игра в нестыковки. Кнопка `[🗣 Сослаться на показания]`.

2. **Театральная развязка (а-ля Пуаро).** При обвинении бот не просто говорит
   «верно/неверно», а **кинематографично разоблачает**: собирает всех «в гостиной»,
   нагнетает, перечисляет предъявленные игроком улики, ломает убийцу. Отдельный
   «театральный» промпт поверх судьи (E). На демо — главный эмоциональный пик.

### В стретч (если останется время)

3. **Динамичный убийца / второй труп.** Если игрок долго копает не туда (потратил
   N ходов без прогресса) — убийца наносит второй удар: гибнет свидетель, доска
   меняется, появляется новая улика. Саспенс и реиграбельность.

4. **Карточка-результат для шаринга.** В финале — картинка «Дело раскрыто за 8 ходов ·
   ранг Пуаро · вселенная Нуар». Репостится → виден «продукт», вирусность.

5. **Живой детектор лжи.** Когда игрок давит близко к `secret`, в ответе подозреваемого
   проступают «теллы» (запинается, потеет, переводит тему) — задаётся правилом в
   промпте C. Атмосфера + честная косвенная подсказка.
