# 🗓️ План разработки «Detective» — 7 дней

> Принцип: **риск и играбельность — раньше, украшательства — позже.**
> Каждый день кончается измеримым результатом. День 7 — буфер + демо.
> Концепт: см. `КОНЦЕПТ.md`.

Легенда: 🎯 цель дня · ✅ к концу дня работает · ⚠️ риск/на что смотреть.

---

## День 1 — Фундамент + убийца рисков (генерация дела)

🎯 Поднять каркас и **сразу закрыть самый большой риск** — умеет ли LLM генерить
решаемое дело.

- [ ] Структура проекта (`bot/ core/ db/ prompts/ config.py`), venv, зависимости
      (`aiogram`, выбранный LLM-SDK, `python-dotenv`).
- [ ] `config.py` + `.env`: токен бота, ключ LLM, переключатель провайдера.
- [ ] `core/llm.py` — обёртка `ask(system, user) -> str` + режим JSON-ответа.
      Свапаемый провайдер (Groq/Gemini → OpenAI одной строкой).
- [ ] Минимальный бот: отвечает на `/start` «привет» (проверка токена/поллинга).
- [ ] **Промпт A (генерация дела)** + `core/case_gen.py`. Гоняем из CLI-скрипта,
      НЕ через Telegram. Получаем валидный JSON `case` по схеме из концепта.
- [ ] **Промпт B (валидатор)** — проверка решаемости, регенерация при дыре.

✅ Из консоли командой генерится корректное, логически решаемое дело (4 подозр.,
3–4 локации, цепочка улик ведёт к убийце). Бот живой на `/start`.

⚠️ Если дело часто кривое/нерешаемое — здесь и только здесь это чинить. Это сердце.

## День 2 — Допрос подозреваемого (ядро геймплея)

🎯 Главная вау-механика: говоришь с подозреваемым живым текстом.

- [ ] SQLite-схема + слой доступа: `users`, `cases`, `sessions`.
- [ ] FSM-состояния aiogram (`menu / playing / interrogating(...)`).
- [ ] Игровой экран: постоянная клавиатура (пока 2 кнопки: Допросить / Обвинить-заглушка).
- [ ] Выбор подозреваемого → вход в диалог.
- [ ] **Промпт C (отыгрыш подозреваемого)** + `core/interrogate.py`: в модель уходит
      только кусок дела этого подозреваемого + короткая память `suspect_dialogs[id]`.
- [ ] Сохранение диалога в сессию (память между репликами).

✅ В Telegram: начал дело → выбрал подозреваемого → задаёшь вопросы своими словами,
он отвечает в характере, помнит предыдущие реплики и НЕ сливает убийцу.

⚠️ Проверить «джейлбрейк»: попросить подозреваемого назвать убийцу — не должен.

---

## День 3 — Улики, осмотр локаций, блокнот, экономика ходов

🎯 Превратить «диалог» в «расследование».

- [ ] **Промпт D (осмотр локации)** + `core/search.py`: свободный текст осмотра →
      сопоставление с `searchables` → выдача улики или «пустышка».
- [ ] Запись найденных улик в `found_clues`.
- [ ] Кнопка `[📋 Блокнот]`: улики / кто допрошен / осталось ходов.
- [ ] Экономика ходов: `moves_left`, списание за осмотр и раунд допроса, стоп при 0.
- [ ] Кнопка `[🎴 Предъявить улику]` внутри допроса → правило `breaks_on` (подозреваемый
      колется на своей улике).

✅ Полный сбор информации: осматриваешь места, находишь улики, давишь уликами на
подозреваемых, видишь прогресс в блокноте, ходы тратятся.

⚠️ Баланс: ~12 ходов должно хватать впритык. Записать, скорректируем на дне 6.

## День 4 — Обвинение, судья, развязка, ранги (полный цикл!)

🎯 Замкнуть игру от старта до финала.

- [ ] FSM `accusing`: выбор убийцы + поле обоснования.
- [ ] **Промпт E (судья финала)** + `core/judge.py`: сверка с `solution_chain`,
      оценка качества обоснования.
- [ ] **Театральная развязка** (изюминка №2): кинематографичное разоблачение.
- [ ] `core/scoring.py`: формула ранга (верность + обоснование + остаток ходов −
      подсказки). Шкала Стажёр→Холмс.
- [ ] Финальный экран + [Сыграть ещё] / [В меню]. Закрытие сессии.

✅ **MVP готов:** можно пройти дело целиком от `/start` до ранга. Это играбельный
продукт — даже если дальше всё сломается, есть что показать.

⚠️ Контрольная точка проекта. Если отстаём — день 5–6 режем стретч, MVP уже есть.

## День 5 — Изюминки + онбординг + выбор вселенной

🎯 Поднять над «чатом с ИИ» и сделать понятным с первого экрана.

- [ ] **Перекрёстный допрос** (изюминка №1): кнопка `[🗣 Сослаться на показания]` —
      в промпт C добавляется цитата другого свидетеля, алиби рассыпаются.
- [ ] Авто-блокнот: добавить трекинг **замеченных противоречий**.
- [ ] Выбор вселенной (Классика / Нуар / Абсурд) → прокидывается в промпт A.
- [ ] Кнопка `[💡 Подсказка]` (снижает ранг).
- [ ] **Туториал**: первое дело — заготовленное мини-дело с подсказками по UI.
- [ ] Пауза/возврат: убедиться, что `[▶️ Продолжить]` поднимает сессию из БД.

✅ Игра ощущается как настоящий детектив (перекрёстный допрос), новичок понимает
правила за минуту, три вселенных, сейв/резюм работают.

## День 6 — Полировка, баланс, деплой, плейтест

🎯 Из «работает у меня» в «не стыдно показать жюри».

- [ ] Плейтест 5–10 партий подряд. Записать все косяки логики/баланса.
- [ ] Баланс: число ходов, сложность лжи, частота пустышек при осмотре.
- [ ] UX-копирайтинг: брифинг, тексты кнопок, финал — атмосферно и без воды.
- [ ] Обработка ошибок: таймаут/сбой LLM, кривой JSON, пустой ввод, ретраи.
- [ ] Оптимизация токенов: проверить, что в модель не утекает лишнее.
- [ ] Деплой: либо стабильный поллинг с ПК, либо VPS + webhook. README + `.env.example`.

✅ Бот стабилен, сбалансирован, читается приятно, развёрнут и доступен по ссылке.

⚠️ Резерв дня — на «вылезшие» баги плейтеста. Стретч НЕ трогаем, пока есть баги.

## Сводка приоритетов (если время горит)

**Нельзя резать (MVP):** генерация+валидация дела, допрос, осмотр/улики, обвинение,
развязка, ранг. → это дни 1–4.

**Сильно желательно:** перекрёстный допрос, туториал, 3 вселенных, баланс. → дни 5–6.

**Можно резать без боли:** дело дня/лидерборд, карточка-шаринга, второй труп, арты. → стретч.

**Контрольные точки:** конец дня 1 — дело генерится; конец дня 4 — игра проходима
целиком; конец дня 6 — стабильно и развёрнуто.
