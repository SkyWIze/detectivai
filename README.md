# 🕵️ Detective — Telegram-бот

Детектив с ИИ-мастером: допрашиваешь подозреваемых живым текстом, осматриваешь
локации, ловишь на лжи и называешь убийцу. Каждое дело генерит LLM.

Концепт — [КОНЦЕПТ.md](КОНЦЕПТ.md), план разработки — [ПЛАН.md](ПЛАН.md).

## Быстрый старт

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
pip install -r requirements.txt

copy .env.example .env   # затем заполни BOT_TOKEN и LLM_API_KEY
python main.py
```

> Без `LLM_API_KEY` бот всё равно запустится — на встроенном демо-деле (`sample`).
> Это удобно, чтобы проверить интерфейс и навигацию без ключей.

## Структура

```
main.py            — точка входа (aiogram, polling)
config.py          — конфиг из .env
bot/
  states.py        — FSM-состояния
  keyboards.py     — клавиатуры
  handlers/        — start.py (меню), game.py (игровой цикл)
core/
  llm.py           — свапаемая обёртка над LLM (Groq/OpenAI/Gemini)
  case_gen.py      — генерация + валидация дела (вызовы A, B)
  interrogate.py   — отыгрыш подозреваемого (C)
  search.py        — осмотр локаций (D)
  judge.py         — финал + театральная развязка (E)
  watson.py        — напарник-компаньон (реагирует на находки и нестыковки)
  scoring.py       — расчёт ранга
  sample_case.py   — демо-дело (туториал/фолбэк)
  models.py        — формы данных
db/database.py     — SQLite (users, cases, sessions, leaderboard)
prompts/           — системные промпты (.txt), правятся без кода
```

## Что готово в скелете
- Запускается, проходится весь флоу на демо-деле: меню → выбор вселенной →
  брифинг → допрос → осмотр → блокнот → обвинение → ранг.
- Все LLM-вызовы вынесены в `core/*` и `prompts/*` — наполняются по плану (дни 1–4).
- Сессии сохраняются в SQLite (пауза/возврат через «Продолжить»).

## Фишки
- **Напарник-компаньон** (`core/watson.py`): живой ИИ-спутник реагирует на брифинг,
  находки улик и пойманные нестыковки. Видит только то, что известно игроку, —
  не спойлит убийцу. Свой характер на каждую вселенную (Ватсон / Сэл / Бубнин).
- **Разнообразие дел**: анти-повтор рамок (соседние дела не дублируют место/жертву/
  конфликт), плавающий размер дела (3–5 подозреваемых, 3–4 локации), «зёрна
  уникальности» и 9 сюжетных твистов — дела перестают быть вариациями одного шаблона.
